# ✨ 贡献指南 (Contributing to Magical Girl Glow Down)

[← 返回主页 (Back to README)](README.md)

感谢你对 **魔法少女·静谧霓虹 (Magical Girl Glow Down)** 的关注！作为一个旨在优化 Windows 桌面 RGB 待机体验的开源项目，我们非常欢迎来自社区的改进、硬件适配建议和 Bug 修复。

由于本项目是一个 **AI 辅助开发 (AI-Augmented Development)** 的项目，我们在接受贡献时会特别关注代码的逻辑清晰度、状态机的健壮性以及硬件访问的安全限制。

## 🛠️ 开发环境配置

本项目使用 [uv](https://github.com/astral-sh/uv) 进行 Python 依赖管理，同时部分硬件通信（如技嘉 SDK 调用）依赖 C# 辅助进程。开发环境要求如下：

1.  **Python 版本**: `3.12`
2.  **.NET SDK**: 安装 .NET SDK（推荐 8.0 或更高版本，用于编译和测试 Gigabyte Helper）。
3.  **安装 uv**:
    ```powershell
    pip install uv
    ```
4.  **同步 Python 依赖**:
    ```powershell
    uv sync --all-groups
    ```
5.  **编译 Gigabyte 辅助进程**:
    ```powershell
    powershell -ExecutionPolicy Bypass -File .\scripts\build-helper.ps1
    ```

## 📝 贡献流程

### 1. 报告 Bug 与新硬件兼容性
*   在提交 Issue 之前，请先搜索现有的 Issue 列表。
*   请提供详细的硬件配置（如主板型号、Nollie控制器版本）、GCC 软件版本、外设连接方式以及调试日志。

### 2. 提交功能建议
*   如果你有新的设想（如支持特定新主板的白名单映射或改进待机检测钩子），请先发起一个 Issue 进行讨论。

### 3. 提交 Pull Request (PR)
*   **分支规范**: 请从 `main` 分支切出你的功能分支（如 `feat/amazing-feature` 或 `fix/bug-name`）。
*   **代码风格**: 本项目使用 `Ruff` 和 `mypy`。在提交前请运行以下命令确保无报错：
    ```powershell
    # 格式化与风格检查
    uv run ruff check . --fix
    uv run ruff format .
    
    # 类型检查
    uv run mypy src
    ```
*   **运行测试**:
    *   Python 测试：`uv run pytest`
    *   C# 辅助进程测试：`dotnet test helper/MagicalGirlGlowDown.GigabyteHelper.Tests -c Release -v minimal`
*   **注释与 AI 声明**: 如果你的代码是由 AI 辅助生成的，请在 PR 描述中注明所使用的模型和主要 Prompt 逻辑，这能帮助我们更快地完成 Review。

## 🏗️ 项目架构简述

*   `src/magical_girl_glow_down/`:
    *   `main.py`: 命令行入口与参数解析。
    *   `tray.py`: 系统托盘 UI、菜单及托盘事件交互。
    *   `service.py`: 调度服务（主状态机），协调空闲检测、进程避让及硬件休眠恢复。
    *   `windows_input.py` / `input_monitor.py`: Windows 键鼠 Hook、XInput 及 Raw Input 手柄活动监听逻辑。
    *   `gigabyte.py` / `gigabyte_helper/`: 技嘉辅助进程通信封装与设备白名单验证。
    *   `protocol.py`: Nollie 控制器 USB HID 协议指令封装与读写。
    *   `app_guard.py`: 官方控制软件（如 GCC、NollieRGB）运行状态监听与主动避让。
    *   `autostart.py`: Windows 任务计划程序免 UAC 提权自启动注册逻辑。
*   `helper/`: 技嘉主板底层的 C# 辅助进程源码与测试。
*   `scripts/`: 包含构建 C# 辅助进程的可执行程序脚本。

# ✨ Contributing Guide (English)

Thank you for your interest in **Magical Girl: Tranquil Neon (Magical Girl Glow Down)**! As an open-source project aimed at optimizing the Windows desktop RGB idle experience, we welcome improvements, hardware mapping suggestions, and bug fixes from the community.

Since this project is developed with **AI-Augmented Development**, we pay special attention to code logic clarity, state machine robustness, and hardware write safety constraints when accepting contributions.

## 🛠️ Development Environment Setup

This project uses [uv](https://github.com/astral-sh/uv) to manage Python dependencies, and certain hardware features (such as Gigabyte SDK integration) rely on a C# helper process. Requirements:

1.  **Python Version**: `3.12`
2.  **.NET SDK**: Install the .NET SDK (8.0 or newer recommended, for building/testing the Gigabyte Helper).
3.  **Install uv**:
    ```powershell
    pip install uv
    ```
4.  **Sync Python Dependencies**:
    ```powershell
    uv sync --all-groups
    ```
5.  **Build the Gigabyte Helper**:
    ```powershell
    powershell -ExecutionPolicy Bypass -File .\scripts\build-helper.ps1
    ```

## 📝 Contribution Process

### 1. Reporting Bugs & Hardware Compatibility
*   Before submitting an Issue, please search the existing Issue list.
*   Provide detailed hardware configuration (e.g. motherboard model, Nollie controller version), GCC version, connection method, and logs.

### 2. Suggesting Features
*   If you have new ideas (such as supporting a new motherboard mapping whitelist or optimizing the idle hooks), please open an Issue to discuss first.

### 3. Submitting Pull Requests (PR)
*   **Branching**: Please create your feature branch from the `main` branch (e.g., `feat/amazing-feature` or `fix/bug-name`).
*   **Code Style**: We use `Ruff` and `mypy`. Run the following before submitting:
    ```powershell
    # Format and Style Checks
    uv run ruff check . --fix
    uv run ruff format .
    
    # Type Check
    uv run mypy src
    ```
*   **Running Tests**:
    *   Python Tests: `uv run pytest`
    *   C# Helper Tests: `dotnet test helper/MagicalGirlGlowDown.GigabyteHelper.Tests -c Release -v minimal`
*   **AI Disclaimer**: If your code was assisted or generated by AI, please state the model used and main prompt logic in the PR description to help us review more efficiently.

## 🏗️ Architecture Overview

*   `src/magical_girl_glow_down/`:
    *   `main.py`: CLI entry and argument parsing.
    *   `tray.py`: System tray UI, context menus, and tray interaction.
    *   `service.py`: Orchestrator service (main state machine), coordinates idle timeouts, app-guard pausing, and sleep/restore triggers.
    *   `windows_input.py` / `input_monitor.py`: Windows keyboard/mouse hook, XInput, and Raw Input game controllers monitoring.
    *   `gigabyte.py` / `gigabyte_helper/`: Wrapper for the C# helper process communication and motherboard whitelist validation.
    *   `protocol.py`: Nollie controller USB HID protocol packet construction and direct I/O.
    *   `app_guard.py`: Detects when vendor softwares (GCC, NollieRGB) are running for automated control pausing.
    *   `autostart.py`: Windows Task Scheduler registration for UAC-free login startup.
*   `helper/`: C# source code and unit tests for the Gigabyte helper process.
*   `scripts/`: Automation scripts including helper build commands.
